## 🤝 如何贡献

感谢你对《从零开始的代数几何生活》的关注！本项目旨在以开源协作的方式，共同打造一份零基础友好的代数几何笔记。

### 📌 如果你是外部读者（非 Perplexity-Studio 成员）
- **欢迎通过 Issues 报告任何错误、拼写问题或提出改进建议**。
- 由于项目尚处于内部建设阶段，暂不接受外部直接提交的笔记。但我们珍视每一条反馈，它将帮助我们做得更好。

### 👥 如果你是 Perplexity-Studio 成员

内部成员可以参与笔记的编写与项目规划。为了保证协作顺畅，我们采用 **“先认领，后开发”** 的两步流程：

---

#### 第一步：认领任务

1. **查看待办计划**  
   浏览仓库根目录下的 `future-plan` 文件夹，里面列出了所有规划中的笔记项目。每个文件（或条目）代表一个待完成的任务。

2. **创建认领分支**  
   从最新的 `main` 分支创建一个新分支，命名格式为 `claim/项目名`（例如 `claim/仿射簇`）。
   ```bash
   git checkout main
   git pull origin main
   git checkout -b claim/仿射簇
```

1. 标记认领
   修改 future-plan 中对应的文件，将状态由“待认领”改为“进行中”，并注明认领人（例如 认领人: @你的用户名）。
   如果计划采用文件方式管理，可以直接在文件中添加认领信息。
2. 提交 Pull Request
   推送分支并创建一个 Pull Request，目标分支为 main。在 PR 描述中简要说明你认领的项目。
   该 PR 仅用于更新 future-plan，不包含任何笔记内容。
3. 合并认领 PR
   PR 需至少获得一位其他成员的审查。通过后，维护者将使用 Squash and merge 方式合并。
   合并后，main 分支上的 future-plan 已更新，其他人便不会重复认领同一任务。

---

第二步：编写笔记

1. 同步最新的 main
   认领 PR 合并后，先将本地 main 分支更新到最新：
   ```bash
   git checkout main
   git pull origin main
   ```
2. 创建开发分支
   基于最新的 main 创建新分支，命名格式为 feat/项目名（例如 feat/仿射簇）。
   ```bash
   git checkout -b feat/仿射簇
   ```
3. 编写笔记
   · 将`future-plan`下的笔记文件放入对应的文件夹中（可参考 example.md 的格式与风格）。 
   · 遵循项目的文档规范（如标题层级、公式写法、引用格式等）完成笔记。
   · 可以分多次提交，逐步完善。
4. 提交 Pull Request
   · 推送分支：git push origin feat/仿射簇
   · 在 GitHub 上创建 Pull Request，目标分支仍为 main。
   · PR 标题应简明扼要，例如 feat: 添加仿射簇笔记。
   · 在 PR 描述中关联第一步的认领 PR（例如 Closes #123），方便追踪。
5. 审查与合并
   · PR 至少需要一位其他成员审查通过。
   · 审查者会检查内容准确性、格式规范性以及是否与项目风格一致。
   · 根据反馈进行修改：只需在本地同一分支继续提交并推送，PR 会自动更新。
   · 合并前请确保分支与最新的 main 无冲突（可执行 git merge main 或 git rebase main 并解决冲突）。
   · 最终由维护者使用 Squash and merge 将笔记合并到 main。

---

📋 注意事项

· 分支命名：认领分支用 claim/ 前缀，开发分支用 feat/ 前缀，便于识别。
· 同步上游：如果你的开发周期较长，请经常拉取 main 的更新，避免后期冲突。
· PR 描述：清晰说明改动内容，认领 PR 只需说明认领项目，开发 PR 则应概述笔记主要内容。
· 合并策略：我们统一使用 Squash and merge，将分支上的所有提交压缩为一个，保持 main 历史整洁。
· 沟通：如有任何疑问，可在 PR 中留言或通过团队内部渠道讨论。
